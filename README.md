# Matrix Game Solver

 Проєкт містить графічний інтерфейс, програмний API для обчислень, підтримку редукції матриці, аналітичні та графічні методи, а також розв'язання загального випадку через симплекс-метод.

## Можливості

- визначення сідлової точки та розв'язання через чисті стратегії;
- спрощення матриці через доміновані рядки та стовпці;
- аналітичний розв'язок для вироджених випадків і матриць 2x2;
- графічні методи для випадків 2xN та Mx2;
- загальний розв'язок через лінійне програмування / симплекс-метод;
- Tkinter GUI для введення матриці з клавіатури або з файлу;
- побудова графіків для доступних випадків;
- збереження детального логу розв'язання у файл;
- бенчмаркінг швидкодії для квадратних матриць різних розмірів.

## Вимоги

- Python 3.10 або новіший;
- `numpy`
- `scipy`
- `matplotlib` 
- `tkinter` 

## Встановлення

1. Клонуйте репозиторій:
```bash
git clone https://github.com/uhorin/matrix-game-solver.git
```
2. Встановіть залежності:

```bash
pip install numpy scipy matplotlib
```

Якщо графіки не потрібні, `matplotlib` можна не встановлювати.

## Запуск

### Графічний інтерфейс

Запустіть головний файл із кореня проєкту:

```bash
python matrix_game/main.py
```

### Бенчмаркінг

```bash
python matrix_game/benchmarking.py
```

Після виконання може бути збережено графік `time_complexity_graph.png`.

## Формат введення

### Введення з клавіатури

- задайте кількість рядків і стовпців;
- заповніть матрицю числами;
- можна використовувати як крапку, так і кому в десяткових значеннях.

### Введення з файлу

Підтримуються файли `.txt`, `.dat` і `.csv`.

Формат файлу:

- один рядок матриці на один рядок файлу;
- числа розділяються пробілами;
- усі рядки мають однакову кількість елементів.

Приклад:

```text
1 2 3
4 5 6
7 8 9
```

## Як працює розв'язання

Логіка вибору методу побудована так:

1. Якщо матриця має один рядок або один стовпець, обробляється вироджений випадок.
2. Для матриці 2x2 запускаються графічний і аналітичний методи.
3. Для випадків 2xN та Mx2 використовується графічний розв'язок.
4. Для загального випадку застосовується симплекс-метод.
5. Перед цим виконується перевірка сідлової точки та редукція домінованих стратегій.

## Структура проєкту

- [matrix_game/main.py](matrix_game/main.py) — точка входу для GUI.
- [matrix_game/gui_app.py](matrix_game/gui_app.py) — Tkinter-інтерфейс, введення матриці, запуск розв'язання, експорт логу.
- [matrix_game/game.py](matrix_game/game.py) — основний клас, який керує вибором методу та формуванням результатів.
- [matrix_game/solvers/](matrix_game/solvers) — модулі з окремими методами розв'язання.
- [matrix_game/visualization/plots.py](matrix_game/visualization/plots.py) — побудова графіків.
- [matrix_game/benchmarking.py](matrix_game/benchmarking.py) — вимірювання швидкодії.
- [matrix_game/formatting.py](matrix_game/formatting.py) — форматування матриць для виводу.

## Примітки

- Якщо `matplotlib` не встановлено, програма продовжить роботу без графіків.
- У GUI є режим детального логу, який можна показати або сховати.
- Для коректного запуску використовуйте кореневу папку проєкту як поточну директорію.
